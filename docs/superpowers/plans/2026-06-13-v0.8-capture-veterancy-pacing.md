# v0.8 — Deliberate Capture, Veterancy, Base Visuals & Simultaneous Replay

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make base capture a deliberate, opt-in choice; reward surviving killers with a repeatable veterancy promotion; refresh base visuals with a capture animation; and re-pace the replay so all moves and all combats resolve simultaneously.

**Architecture:** Four independently-shippable phases. Phase 1 (veterancy) and Phase 2 (deliberate capture) are pure-core changes (TDD-strict, `npm test` gated). Phase 3 (base visuals + capture FX) and Phase 4 (simultaneous pacing) are UI/`src/state` changes verified by build + visual check. Phases are ordered so later phases absorb the new events earlier phases emit: do them in order, and don't start phase N+1 with phase N red (CLAUDE.md / spec §12).

**Tech Stack:** TypeScript, Vite, React, Vitest, Zustand. Pure determinism enforced by `scripts/check-purity.mjs` (runs first in `npm test`). `src/board`, `src/core`, `src/ai` must stay free of `Math.random`, `Date.now`, argless `new Date()`, DOM, and `ui/`/`state/` imports.

## Design Decisions (locked with the operator 2026-06-13)

These resolve ambiguities in the raw request. Log them to `.deban/` after Phase 1 lands.

1. **Capture is opt-in.** Today every personnel unit on an unowned base captures automatically and is consumed (`resolver.ts:529–548`). New rule: a unit captures **only if it carries a `capture` order** this round. No order → it stands on the base, survives, flips nothing. Default is **off** (deliberate). The claim still **consumes** the unit (v0.6 rule unchanged).
2. **Capture is a 4th order kind**, `{ kind: 'capture'; unitId }`, slotting into `UnitOrders` exactly like stance/move/attack (one per unit). This covers both "move onto a base and claim" and "claim the base I already stand on." A capture order on a unit that doesn't end on a capturable base is a harmless no-op.
3. **The AI must emit capture orders** or it stops capturing (regression of the conquest AI). The planner adds a capture order whenever it routes a personnel unit to an enemy/neutral base it intends to take.
4. **Veterancy XP** accrues to the unit whose strike reduces an enemy to count 0. `xp += round(0.10 × killedType.cost)` (full type cost — "the value of the unit killed"). XP accrues on the kill regardless of whether the killer later dies.
5. **Promotion is end-of-round, survivors only**, repeatable. After capture (Phase B.5), before `attackedFrom` is cleared: for each alive unit, `newRank = floor(2 × xp / cost)`. If `newRank > rank`, heal `2 × (newRank − rank)` count (capped at 10) and set `rank = newRank`. Dead/consumed units are already removed, so they never promote — "killed **and survived**."
6. **The damage bonus is `+rank` to attack strength**, applied from the round *after* promotion (promotion happens end-of-round). It applies in Phase B attacks, counters, and Phase A.5 brawls. It only applies where base attack strength is already > 0 (rank never lets a unit damage an armor type it couldn't touch).
7. **Veterancy is visible**: rank pips on the unit token (planning + replay) plus a promotion flair when a unit ranks up during replay.
8. **New `UnitInstance` fields `xp` and `rank` are optional** (`xp?: number; rank?: number`), read as `?? 0`. This avoids churning every unit-literal in the test suite; new units initialise both to 0 at their real construction sites.

```ts
export function explainAttack(ctx: AttackContext): AttackTerms {
  const { attacker, defender, bonusB } = ctx;

  const baseA = attacker.type.attackStrengths[defender.type.armorType];
  // Veterancy only sharpens an attack that can already land (base A > 0).
  const vet = baseA > 0 ? (attacker.damageBonus ?? 0) : 0;
  const A = baseA + vet;
  const D = defender.type.armor;
  const Ta = attacker.type.terrainEffects[attacker.terrain]?.attackBonus ?? 0;
  const terrainTd = defender.type.terrainEffects[defender.terrain]?.armorBonus ?? 0;
  const stanceTd = defender.stance === 'defensive' ? 1 : 0;
  const Td = terrainTd + stanceTd;

  if (attacker.count <= 0 || defender.count <= 0 || A <= 0) {
    return { A, Ta, D, Td, B: bonusB, vet, p: 0, damage: 0 };
  }

  const p = clamp01(0.5 + 0.05 * (A + Ta - (D + Td) + bonusB));
  const engagements = Math.min(attacker.count, defender.count);
  const raw = roundDamage(engagements * p);
  const damage = raw === 0 && p > 0 ? 1 : raw;
  return { A, Ta, D, Td, B: bonusB, vet, p, damage };
}
```
